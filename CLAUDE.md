# 《仙路烟尘》创作项目守则

这是长篇修仙小说《仙路烟尘》的创作项目。本文件是每次会话的最高行为准则。

## 项目性质

- 凡人流修仙长篇，100 万字以上，每章 4000~6000 字。
- 全书设计依据：`docs/specs/2026-06-12-xianlu-yanchen-design.md`（设定冲突时以它为准，改设定先改它）。
- **文件是唯一事实来源**：不依赖会话记忆，一切创作状态以本仓库文件为准。

## 会话启动协议（写作前必读）

按顺序装载，缺一不可：

1. `bible/style-guide.md` — 文风纪律
2. `bible/outline-vol<当前卷>.md` — 当前卷卷纲与章纲
3. `bible/plot-threads.md` — 伏笔台账（重点看"未回收/推进中"）
4. `bible/characters/_index.md` — 人物总览，再装载其中标记 active 的人物全档
5. `summaries/` 最新卷摘要 — 快速回放剧情
6. `chapters/` 最近 2~3 章全文 — 接续文气

inactive 人物只读 _index 里的一行摘要；dead 人物档案不装载。**禁止删除任何人物档案，只改状态。**

## 记账协议（每写完一章必做，不可跳过）

1. `summaries/v<卷>.md` 追加本章 100~200 字摘要
2. `bible/plot-threads.md` 登记新埋伏笔 / 更新推进状态 / 标记已回收
3. `bible/characters/` 更新出场人物的状态与变化（含 active/inactive/dead 流转）
4. `bible/timeline.md` 登记时间流逝与大事
5. git 提交本章及档案变更

## 写作纪律（红线）

- 修为增速严格按卷纲核定值，严禁超速；资源有价，危险真能死人。
- 手札只给经验不给力量；手札信息可因过时而坑主角。
- 主角视角叙事，禁止全知视角剧透暗线。
- 每章末必须有钩子；"恰好"式好运全书限量。
- 伏笔台账"未回收"超过 15 条时，先收再埋。
